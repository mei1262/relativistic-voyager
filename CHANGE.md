# Relativistic Voyager Alpha

## 钟慢效应与尺缩效应 — 形象直观体现方案与视觉效果规格书

**文档版本：** 2026-07-07
**适用项目：** relativistic-voyager-alpha1
**技术栈：** Three.js + WebXR + Vite
**目标读者：** 开发者、教学设计者、VR 体验策划

## 文档说明

本文档将狭义相对论中的**钟慢效应**与**尺缩效应**，转化为在本项目中可落地、可感知、可教学的**具体视觉方案**。每个方案均包含：
- 用户看到什么（视觉效果描述）
- 物理含义（避免误解）
- 触发条件（β、参考系、显示模式）
- 实现要点（对应代码模块与参数建议）
- 教学用途（适合哪类任务/讲解）
另含**星空光学效应**完整规格：前方蓝移聚集、后方红移稀疏、隧道感等。

### 1.2 核心差距（用户「看不见」的原因）

- **MeasurementRod 未在 `App.js` 中实例化**，任务 3「观察测量杆收缩」无法完成视觉层面。
- **`frame` 与 `viewMode` 仅影响 HUD 徽章和时空图**，飞船 mesh、星场、距离标牌不随参考系变化。
- **星场 `update()` 明确忽略 β**，前方聚集、蓝移、后方红移均未实现。
- **钟慢仅有静态数字**，缺少「两钟走时速度不同」的动态感知。

### 1.3 设计总原则

| 原则 | 说明 |
| --- | --- |
| 分层呈现 | 数字（HUD）→ 物体（杆/飞船）→ 环境（星空）→ 叙事（里程碑）四层叠加 |
| 参考系一致 | 同一画面只表达一个参考系的 measured 结果，切换 frame 时场景跟着变 |
| 测量 ≠ 观察 | Measured 与 Observed 必须有可感知的视觉差异（颜色、旋转、标签） |
| 方向性 | 尺缩只沿运动方向；光学效应前后不对称 |
| 教学优先 | 宁可略失真也要「一眼看懂」，进阶版再做 Penrose-Terrell 精确渲染 |

### 2.2 方案 T1：双钟并排走时动画（首选）

#### 视觉效果描述

屏幕上方或 HUD 面板内并排两个模拟数字钟：

```text
┌─────────────────────┐  ┌─────────────────────┐│   🌍 地球钟 t       │  │   🚀 飞船钟 τ       ││   12.847 年         │  │   5.592 年          ││   ▓▓▓▓▓▓▓▓░░  72%   │  │   ▓▓▓░░░░░░  31%   │└─────────────────────┘  └─────────────────────┘         走得更快                    走得较慢
```

- β=0 时两钟完全同步。
- β=0.866（γ=2）时，地球钟每跳 2 个单位，飞船钟跳 1 个。
- β=0.99 时差距极大：地球钟飞速滚动，飞船钟缓慢爬行。

#### 物理含义

- 地球钟显示坐标时 earthTime（地球参考系累积时间）。
- 飞船钟显示固有时间 τ = earthTime / γ。
- **不是**船头钟和船尾钟不同，而是**地球系与飞船系的时间定义不同**。

#### 触发条件

- 飞船前进（currentSpeed > 0）且未暂停时，两钟同时走动。
- 暂停时冻结；β=0 时两钟速率相同。

#### 实现要点

```js
// 每帧更新（伪代码）const r = computeRelativityState(state);earthClockDisplay = r.earthTime;shipClockDisplay = r.shipTime;// 动画：数字跳动使用 requestAnimationFrame 插值// 地球钟 tick 间隔 = baseInterval// 飞船钟 tick 间隔 = baseInterval * gamma  （即慢 gamma 倍）
```

- 新增文件建议：src/ui/DualClockPanel.js
- 在 index.html HUD 区增加 #earth-clock-canvas 与 #ship-clock-canvas
- 可选：飞船钟滴答声，音调随 β 降低（艺术化提示，非物理必需）

#### 教学用途

- 任务 2「观察时间膨胀」的核心视觉载体。
- 课堂演示：先 0.1c 两钟重合，再拉到 0.9c 看分化。

### 3.2 方案 L1：双测量杆（平行 vs 垂直）—— 尺缩教学金标准

#### 视觉效果描述

飞船旁漂浮两根发光测量杆，带刻度：

```text
杆 B（垂直运动方向）           │           │  长度不变           │    ═══════════════  杆 A（平行运动方向）       变短 ←→
```

- β=0：两杆等长（例如均为 5.2 单位）。
- β=0.9，Earth + Measured：杆 A 缩至约 44%（1/γ≈0.436），杆 B **不变**。
- 杆 A 颜色：金黄 #ffd36b（Measured）/ 淡蓝 #88c7ff（Observed）。

#### 物理含义

- 尺缩只发生在**沿相对运动方向**的测量长度。
- 垂直方向的长度在各参考系中一致（横向不变）。
- 这是区分「各向同性压扁」误解的最有效演示。

#### 触发条件

| frame | viewMode | 杆 A（平行 -Z） | 杆 B（垂直 X） |
| --- | --- | --- | --- |
| earth | measured | scale.z = 1/γ | 不缩放 |
| earth | observed | 近似收缩 + 微旋转 | 不缩放 |
| ship | measured | 不缩放（共动系） | 不缩放 |
| ship | measured | HUD 显示外界距离缩短 | — |

#### 实现要点

- App.js → setupScene()：

```js
import { MeasurementRod } from '../visual/MeasurementRod.js';this.rodParallel = new MeasurementRod();this.rodParallel.group.rotation.y = -0.25; // 沿 -Zthis.rodParallel.addTo(this.scene);this.rodPerpendicular = new MeasurementRod();this.rodPerpendicular.group.rotation.y = -0.25 + Math.PI / 2; // 沿 Xthis.rodPerpendicular.labelSprite // 改标签为「⊥ 运动方向」this.rodPerpendicular.addTo(this.scene);
```

- update() 中：

```js
const applyContraction = (state.frame === 'earth');this.rodParallel.update(  applyContraction ? beta : 0,  state.viewMode);this.rodPerpendicular.update(0, state.viewMode); // 永不受尺缩
```

- 杆跟随飞船：setReferencePosition(shipX, shipY, shipZ)

#### 教学用途

- 任务 3 核心视觉。
- 讲解时先问：「你觉得两根杆都会变短吗？」再加速揭示答案。

### 3.3 方案 L2：地球系飞船外形收缩

#### 视觉效果描述

第三人称追尾相机下，飞船沿 -Z 飞行：
- β=0：正常比例，流线型飞船。
- β=0.9（Earth + Measured）：飞船沿飞行方向明显「压扁」，像被从前后挤压；**翼展/高度不变**。
- β=0.99：飞船呈薄饼状，但仍可辨认。
切换至 Ship 帧：飞船恢复饱满外形（船员看自己船，永远是静止长度）。

#### 实现要点

```js
const base = 0.12; // 现有 scaleconst ratio = lengthContractionRatio(beta);if (state.frame === 'earth' && state.viewMode === 'measured' && beta > 0.01) {  // 模型 nose 朝 -Z，速度沿 -Z  spacecraft.group.scale.set(base, base, base * ratio);} else {  spacecraft.group.scale.setScalar(base);}
```

- **仅 scale Z 轴**（运动方向），不 scale X/Y。
- Observed 模式：额外 rotation.x += beta * 0.3 模拟 Terrell 旋转感。

#### 视觉对比示意（文字）

```text
β=0        β=0.9 measured      β=0.9 observed  ▶──          ▶─               ▶─↗（略倾斜、泛蓝）正常           压扁              不完全压扁
```

## 第四章  星空光学效应 — 完整视觉效果方案

钟慢和尺缩是「测量效应」；窗外星空变化是用户**最强烈的主观感受**。
前方蓝移聚集、后方红移稀疏，是近光速旅行的标志性视觉语言。

### 4.1 效应总览

| 效应 | 物理公式 | 视觉表现 | 感知强度 |
| --- | --- | --- | --- |
| 相对论光行差 | cosθ'=(cosθ+β)/(1+βcosθ) | 星场向船头聚集，船尾出现空洞锥 | ★★★★★ |
| 相对论多普勒 | f'/f=γ(1+βcosθ) | 前方蓝白亮，后方暗红 | ★★★★★ |
| 亮度集中 | 行差+多普勒综合 | 船头像探照灯照亮的前方 | ★★★★ |
| 隧道效应 | 主观+暗角 | 周边漆黑，前方光亮通道 | ★★★★ |
| 目标星多普勒 | 正前方 cosθ≈1 | 比邻星极度蓝亮、膨胀 | ★★★ |

### 4.2 光行差（Stellar Aberration）视觉规格

#### 视觉效果描述

- **β=0**：星空各向均匀，银河带清晰可见。
- **β=0.5**：星星轻微向船头（-Z）偏移，船尾星星略稀疏。
- **β=0.9**：明显「流星隧道」——绝大多数亮星挤在前方 30° 锥角内；船尾 120° 区域几乎无星。
- **β=0.99**：极端前向光柱，两侧全黑，像冲入一条星光隧道。

#### 参数表

| β | 前方聚集锥角（半角约） | 船尾暗锥半角 | 星密度前方/后方比 |
| --- | --- | --- | --- |
| 0.3 | ~40° | 无明显 | 1.5:1 |
| 0.7 | ~25° | ~60° 稀疏 | 4:1 |
| 0.9 | ~15° | ~80° 几乎空 | 12:1 |
| 0.99 | ~5° | ~95° 空 | 50:1+ |

#### 实现算法（每颗星）

```js
// 对星方向单位向量 starDir，速度沿 -Z（船头）const cosTheta = -starDir.z; // +1 = 正前方const cosThetaPrime = aberratedCosTheta(beta, cosTheta);// 将 starDir 映射到新方向 starDirPrime// 简化：只调整 z 分量并 renormalize，或用球面插值
```

#### 实现路径

| 阶段 | 方法 | 文件 |
| --- | --- | --- |
| Phase 1 | CPU 每帧更新 BufferGeometry 位置（限 β>0.1 时） | `StarField.js` |
| Phase 2 | 自定义 `ShaderMaterial`，uniform `beta` | `src/visual/shaders/starfield.*` |
| Phase 3 | 立方体贴图 + 后处理径向模糊 | 进阶 |

#### 性能注意

- 8000 星 CPU 更新：β 变化时批量算，静止 β 时缓存。
- Quest VR：星数降至 4000 或全 GPU。

### 4.3 多普勒颜色偏移（Doppler Shift）视觉规格

#### 视觉效果描述

- **正前方（cosθ=+1）**：星星变为**蓝白色**，亮度增强，尺寸略放大。
- **正后方（cosθ=−1）**：星星变为**暗红色**，亮度减弱，部分几乎不可见。
- **侧方（cosθ≈0）**：颜色轻微黄移，变化温和。
- **β=0.99 前方**：前方星场呈**青蓝色光墙**，刺目、饱和。

#### 颜色映射表（教学用近似）

| f'/f 比值 | 颜色 RGB 建议 | 语义 | 典型位置 |
| --- | --- | --- | --- |
| > 3.0 | (180, 220, 255) 亮青蓝 | 极强蓝移 | 正前 β>0.9 |
| 1.5–3.0 | (140, 180, 255) 蓝白 | 强蓝移 | 前锥 30° |
| 0.8–1.5 | (255, 255, 240) 近白 | 轻微偏移 | 侧方 |
| 0.3–0.8 | (255, 160, 80) 橙红 | 红移 | 后侧 |
| < 0.3 | (120, 40, 30) 暗红 | 极强红移 | 正后 β>0.9 |

#### 亮度映射

```js
const doppler = dopplerFactor(beta, cosTheta);const brightness = clamp(doppler, 0.15, 4.0);const sizeMultiplier = clamp(Math.sqrt(doppler), 0.3, 2.5);starColor.rgb *= brightness;starPointSize *= sizeMultiplier;
```

### 4.4 比邻星（目标恒星）专属效果

#### 视觉效果描述

- 目标星位于 -Z 远方（z = -3000），始终在航线尽头。
- β 增大时：
- 目标星**视觉张角增大**（Observed 近似，非 measured 缩放）。
- 颜色从淡黄 → 青白 → 近乎纯白刺目。
- 外围光晕（glow sprite）半径随 β 增大：glowScale = 1 + beta * 3。
- β=0.99 时：目标星占视野约 15–20°，像「冲进的太阳」。

#### 实现要点

- 修改 SceneObjects.js 中 targetGroup 的 update(beta) 方法
- targetMat.color 按多普勒偏移
- glowSprite.scale 动态调整

### 4.8 β 分档视觉预设（设计稿）

#### 档位 0：低速巡航（β < 0.2）

- 星空：基本静态，轻微闪烁。
- 颜色：原色。
- 钟慢：γ<1.02，双钟几乎同步。
- 尺缩：1/γ>0.98，杆 A 几乎看不出变化。
- 隧道暗角：关闭。

#### 档位 1：亚相对论（β = 0.2–0.6）

- 星空：轻微前向偏移，侧方星略增亮。
- 颜色：前方微蓝，后方微红（subtle）。
- 钟慢：双钟开始分叉。
- 尺缩：杆 A 可见缩短。
- 暗角：opacity 0.2–0.4。

#### 档位 2：相对论显著（β = 0.6–0.9）

- 星空：明显隧道，前 30° 星密度极高。
- 颜色：前方蓝白，后方橙红暗淡。
- 钟慢：地球钟明显快于飞船钟。
- 尺缩：飞船（Earth 帧）明显变扁。
- 暗角：opacity 0.6–0.8。
- 音效：引擎音调升高。

#### 档位 3：近光速（β = 0.9–0.99）

- 星空：极端前向光柱，后方漆黑。
- 颜色：前方饱和青蓝，目标星如探照灯。
- 钟慢：差距巨大（γ>2 到 22）。
- 尺缩：飞船薄饼状 / 杆 A 仅剩 20–5%。
- 暗角：opacity 0.9+。
- 隧道感：FOV 收窄 + 全屏前方泛蓝薄雾（可选 post-processing）。
